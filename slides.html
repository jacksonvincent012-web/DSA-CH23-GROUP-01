DSA: StockHashMap

What We Needed

O(1) stock lookup by ticker symbol across 10,000 stocks

What We Considered

  • Sorted array + binary search — O(log n) but O(n) insert ❌
  • Binary search tree — O(log n) avg, could degrade ❌
  • Python dict (hash table) — O(1) avg ✅

Key Takeaway

Hash table first for O(1) lookup — but handle missing keys gracefully

Verification

DSA: IngestionQueue & AlertStack

📦 IngestionQueue (FIFO)

What we needed: Buffer price ticks without blocking API

What we considered: list.pop(0) = O(n) ❌ → deque.popleft() = O(1) ✅

Lesson: Never use list as a queue — deque is O(1) on both ends

Nenqueuedequeue
1K771μs513μs
10K1,567μs1,887μs

✅ O(1) confirmed

📚 AlertStack (LIFO + Undo)

What we needed: Create alerts, undo last deletion

What we considered: List + undo buffer — simple, O(1) push/pop

Lesson: Stack undo is elegant — one secondary buffer stores the last popped item

DSA: TopKHeap & SectorGraph

📈 TopKHeap (Min-Heap)

What we needed: Top 5 stocks by volume without sorting 10K

What we considered: Full sort = 50ms ❌ → Heap O(N log K) = 0.3ms ✅

Lesson: 170× faster by maintaining candidates incrementally

K=10 O(log K)

🔗 SectorGraph (BFS/DFS)

What we needed: "Which sectors are related to TECH?"

What we considered: Adjacency list — O(V+E) traversal

Lesson: BFS gives closest neighbours first; DFS gives full chain. Different questions, same graph.

DSA: LRU Cache (Bonus)

What We Learned

  • 80/20 rule: 80% of requests hit 20% of stocks
  • HashMap + Doubly Linked List = O(1) get/put
  • Pointer surgery is tricky — wrong node eviction bug caught by tests
  • 66.7% hit rate after just 3 requests in testing

Stats

Capacity: 50 · Size: 1 · Hits: 2 · Misses: 1

✅ O(1) confirmed — flat across 1K–100K

Architecture

Client → GET /api/stocks/AAPL
  → LRUCache.get("AAPL")
    → HIT → return cached ✅
    → MISS → StockHashMap.get()
      → LRUCache.put("AAPL", result)
      → return result
